Phase 1 ‚Äì User Management (Core Foundation)

  • User creation (normal & superuser)
  • Role assignment

2. Admin Panel Integration

Register User model in Django admin. Add custom list display, search, and filters for roles.

Write admin tests:

  • Confirm user appears in admin
  • Role visibility
Phase 2 ‚Äì Authentication & Authorization

1. Authentication Endpoints

Implement token or session-based login and registration.

TDD tests for:

  • Successful registration
  • Duplicate email errors

  • Incorrect credentials handling

2. Role-Based Permissions

Define custom permission classes in DRF:

  • Student-only views
  • Instructor-only views
  • Admin-only views

TDD tests for:

  • Correct access for each role
  • Denied access for incorrect roles
